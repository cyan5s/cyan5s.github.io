Druid未授权

Created2025-11-07|Updated2025-11-22|WEB
|Post Views:

Druid是什么

  • 一款查询、分析的大数据引擎，通过SQL接口、API接口、Web控制台访问——正常会有身份验证来限制未授权访问

未授权漏洞

  • 公网访问敏感API或管理界面

    • 直接访问

    • 绕过身份验证（例如登录、令牌等）访问

Druid未授权漏洞

根本原因

  • 在默认配置下允许任意访问

  • 通过插件实现安全功能（认证），配置不当会导致返回默认配置

  • 即使认证，若管理端口暴露公网会增大攻击面；前置反向代理配错会暴露内网管理路径

技术机制

  • 当 HTTP 请求到达 Druid 时，系统会识别请求来源来匹配，若没有用匹配成功，且没有配置强制拒绝逻辑，系统可能返回允许或默认，即任意访问

  • 若用户角色与对应资源映射文件未配置，则普通或匿名用户可访问敏感资源

  • Druid 内部存在服务间的“受信任调用”机制，用于提高权限，配置不当可提权

  • 未鉴权的 API 可能会返回数据源列表、表结构、内部主机名或 JDBC 配置等元数据；这些信息能被用来进一步攻击（横向移动或连接到后端存储）。
    不懂

DruidFOFA指纹（精准->宽泛）

DruidNday漏洞

  • CVE-2019-10089 - Druid HTTP 端口的未经授权访问

  • CVE-2021-25656 - Druid Console 中的权限提升漏洞

    • Druid 控制台的权限控制缺陷，攻击者可通过普通用户权限执行管理员操作。

Druid漏洞评估

  • 在FOFA指纹搜索后，一般会出现以下几个情况：

    • Druid Stat Index

      • 不用登录，直接进入

      • 跳转到另一个页面登录

      • 点击后

    • Druid Monitor

      • 需要验证，一般弱口令

  • 在进入主界面后，主要关注

    • URL监控

      • 获取不同功能界面的路径，如后台

      • 测试未授权越权（？

    • Session监控

      • 可以获得登录的Session

  • 若登录成功，可寻找可以上传的点，如文件上传

  • 根据功能推测，如大道至简，druid弱口令+接口测试组合拳轻松拿下30w+敏感信息 - FreeBuf网络安全行业门户
