前期准备

下载&安装

先下载unity hub，方便后续集中管理和安装多个 Unity 版本

下载链接：Start Your Creative Projects and Download the Unity Hub | Unity

以windows为例，上面点不了就点下面的

安装完成后打开，点击左边一栏的 Installs ，再点击右上角的 Installs Editor ，选择你想要的unity版本（可以先问一下AI你想制作的游戏使用哪个版本更好，当然，若是初学练习，能用就行）

我下载的是Unity 2023.3 LTS

基础了解

对于一个零基础的小白，熟悉一下基本配置还是必要的，但稍微了解一点就够了，毕竟死记硬背半天不如自己操作一遍

首先，我们看到界面分为了六大块：

每个面板的功能及基本操作：

  1. Hierarchy（层级）

    • 位置：通常在左侧
    • 作用：显示场景中的所有对象
    • 操作：点击对象名称可选中它

  2. Scene（场景视图）

    • 位置：通常在中间
    • 作用：可视化编辑场景
    • 操作：
      • 鼠标中键拖动：移动视图
      • 鼠标右键拖动：旋转视图
      • 滚轮：缩放视图

  3. Game（游戏视图）

    • 位置：与Scene视图在同一区域（标签切换）
    • 作用：显示游戏运行时的画面
    • 操作：点击Play按钮后在这里看到游戏

  4. Inspector（检查器）

    • 位置：通常在右侧
    • 作用：显示和修改选中对象的属性
    • 操作：在这里调整各种参数

  5. Project（项目）

    • 位置：通常在底部
    • 作用：显示所有资源文件
    • 操作：双击打开，右键创建新资源

  6. Console（控制台）

    • 位置：通常在底部（与Project在同一区域）
    • 作用：显示程序运行信息和错误
    • 操作：双击错误信息可定位到代码

大概过一遍，如果实在觉得有些抽象，可以去看unity官方的中文教程，每个视频很短，也很有趣，很适合小白。
指路：https://learn.u3d.cn/tutorial/MiniGameDev?chapterId=63562b28edca72001f21d107&sectionId=613192b177062a0020b6b7ec

看前7集就OK了，有一个整体的认识，接着自己从零开发一个小游戏，实现一些基本的功能，就会使用软件了，完全不需要背

零基础游戏开发

通俗易懂点就是让AI帮你写游戏，并教你如何操作

构思

首先，你要知道自己要做一个什么类型的游戏。不过，既然是练手，我更推荐先学习再创新，就找一款市面上的简单单机小游戏来启蒙吧，像贪吃蛇……

由于每次和朋友玩“死神VS火影”都能乱打不腻（没有技巧全凭一腔热血），我决定复刻它
【茅屋为秋风所破版】

策划

《Soul Scroll》前期游戏策划案 (MVP版本)
核心目标： 使用Unity 2022.3.62f1c1的Universal 2D模板，实现《死神VS火影》最核心的“单机双键盘对战”功能。

  1. 核心玩法概述 (Gameplay Loop)
    模式： 1v1单机对战模式。
    玩家： 两名玩家（P1 & P2）在同一台电脑上，通过不同的键盘按键进行对战。
    目标： 通过攻击削减对方的生命值（HP），在限定时间内将对方HP降至0或以下的一方获胜。
    核心循环： 移动 → 攻击 → 技能 → 命中判定 → 得分/胜负判定 → 新一轮对战。
  2. 游戏功能列表 (优先级排序)
    Phase 1: 最核心系统 (必须首先实现)
    [ ] 场景系统: 一个简单的静态背景对战平台。
    [ ] 角色控制系统 (双输入): 实现两套独立的键盘输入控制两个游戏对象。
    [ ] 物理运动系统: 左右移动、跳跃、下蹲。
    [ ] 战斗系统 (基础):
    普通攻击（近战）
    技能攻击（远程，消耗能量）
    受击判定、伤害计算
    [ ] 状态系统: 生命值（HP）、能量值（MP/气）。
    [ ] UI系统: 实时显示P1和P2的HP条、MP条、头像。
    Phase 2: 核心体验深化
    [ ] 战斗系统 (扩展):
    连招系统（Combo）：通过特定顺序的按键形成连击。
    击退/硬直效果：受击后产生的后退和短暂无法操作的状态。
    闪避/瞬步（Dash/Flash Step）：快速位移技能。
    [ ] 角色面向: 角色会根据移动方向自动切换面向（左/右）。
    [ ] 胜负判定逻辑: 回合倒计时、胜负条件判断（HP先归零、时间到后HP多者胜）。
    Phase 3: 内容与优化
    [ ] 美术资源替换: 将程序 placeholder（如方块人）替换为正式的角色Sprite、动画。
    [ ] 音效系统: 添加攻击音效、受击音效、背景音乐。
    [ ] 更多角色: 设计第二个、第三个角色，拥有不同的技能和属性。
  3. 输入控制方案 (Key Part!)
    这是实现双键盘操作的核心。强烈建议使用Unity的新输入系统（Input System Package），它完美支持多玩家输入和按键重映射。
    | 操作功能 | P1 按键 | P2 按键 | 备注 |
    | ：— | :— | :— | :— |
    | 向左移动 | A | Left Arrow | |
    | 向右移动 | D | Right Arrow | |
    | 跳跃 | W | Up Arrow | |
    | 下蹲 | S | Down Arrow | |
    | 普通攻击 | J | Numpad1 | 或 [小键盘 1] |
    | 技能攻击 | K | Numpad2 | 或 [小键盘 2] |
    | 闪避/瞬步 | L | Numpad3 | 或 [小键盘 3] |
    | 大招（预留） | U | Numpad4 | 或 [小键盘 4] |
    为什么用新输入系统？
    解耦： 输入处理与游戏逻辑分离，代码更清晰。
    多设备支持： 天生支持多个输入设备，完美契合双键盘。
    重绑定： 未来轻松实现按键自定义功能。
  4. 角色设计 (Placeholder版本)
    初期先用简单的几何体（如方形、圆形）代表角色，专注于逻辑实现。
    角色A (测试用):
    HP: 100
    MP: 100 (每秒自动回复5点)
    普通攻击: 造成5点伤害，无消耗。
    技能攻击: 发射一个飞行道具，造成15点伤害，消耗20点MP。
    角色B (测试用):
    HP: 120
    MP: 80 (每秒自动回复8点)
    普通攻击: 造成7点伤害，无消耗。
    技能攻击: 进行一次小范围的重劈，造成20点伤害，消耗25点MP。
  5. 技术架构与Unity实现思路
    场景结构:
    GameManager(空物体，单例模式): 总管游戏状态、胜负判定、时间管理。
    InputManager(空物体): 负责处理新输入系统的输入事件，并转发给对应角色。
    P1_SpawnPoint, P2_SpawnPoint(空物体): 角色出生点。
    Canvas: 包含P1、P2的HP/MP条、头像等UI元素。
    角色预制体结构:
    Player(父物体，挂载Rigidbody2D和BoxCollider2D)
    Graphics(子物体，用于挂载SpriteRenderer和Animation)
    AttackPoint(子物体，空物体，用于定位攻击判定的位置)
    核心组件脚本:
    PlayerInputHandler.cs: 接收来自InputManager的输入指令，转换为角色可调用的方法（如 Move(), Jump(), Attack()）。
    PlayerController.cs: 处理角色的移动、跳跃物理逻辑。
    PlayerCombat.cs: 处理攻击逻辑、伤害计算、生成飞行道具。
    Health.cs: 处理生命值、受击、死亡。
    Energy.cs: 处理能量值的管理和回复。
    UIDisplay.cs: 控制UI条的更新。
  6. 开发里程碑 (Milestone)
    Week 1-2: 输入与移动
    导入新输入系统包。
    创建P1和P2的输入控制映射。
    实现两个方块角色的左右移动和跳跃。
    实现角色自动面向。
    Week 3-4: 战斗与状态
    实现攻击动画事件和攻击碰撞体检测（使用BoxCast或OverlapCircle）。
    实现HP和MP系统，并连接到UI。
    实现受击击退效果。
    Week 5-6: 打磨与体验
    实现简单的连招逻辑（如：AAA → 轻击三连）。
    实现闪避技能（无敌帧、快速位移）。
    完成回合计时和胜负判定。
    添加音效和初步美术资源。

就比如~（仅供参考）

But，打开unity项目时发现报错

别急，这是因为AI创建的C#脚本文件还没有被Unity编译（没有依赖），正确的流程应该是先安装依赖再写脚本

解决步骤：

  • 点击 “Enter Safe Mode” 按钮，进入安全模式
  • 在Project窗口中，找到 Assets/Scripts 文件夹，右键点击，选择 Show in Explorer（在资源管理器中显示）
  • 暂时将所有 .cs 文件移动到其他地方
  • 返回Unity
    • 若unity没有自动修复：
      • 点击菜单 Assets → Refresh
      • 点击Unity右上角的 Exit Safe Mode，退出安全模式

修复成功，正常打开页面

AI辅助开发注意事项

踩坑记录

  1. 有些文件编译了第一遍后，可能编译有误，未能实现想要的功能，AI说它又写了一个相同的文件，但编译器还是认为是之前的文件，故仍然无法编译

    • 法1：直接删除该文件，让AI重新写。注意：与该文件关联的功能可能需要重新配置
    • 法2：修改完文件后（如原先有生成文件要先删除），选择另外一种方式编译（如有新生成文件记得删除），再选择之前的方式编译
