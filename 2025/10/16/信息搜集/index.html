常用字段
字段名 说明 示例 用途
ip 根据 IP 地址搜索 ip="123.123.123.123" 查找指定 IP 的资产
domain 根据域名搜索 domain="fofa.info" 查找指定根域名的所有子域名资产
host 搜索包含该字符串的域名或 URL host=".gov.cn" 搜索所有包含 .gov.cn的域名
title 根据网页标题搜索 title="后台管理" 搜索标题为“后台管理”的页面
header 根据 HTTP 响应头搜索 header="thinkphp" 搜索响应头中包含 “thinkphp” 的服务
server 根据服务器信息搜索 server=="nginx" 搜索使用 nginx 作为服务器的资产
cert 根据证书内容搜索 cert="google.com" 搜索 SSL 证书中包含 “google.com” 的资产
body 根据网页正文内容搜索 body="忘记密码" 搜索正文中包含“忘记密码”的页面
port 根据端口号搜索 port="3306" 搜索开放了 3306 端口的资产（MySQL）
protocol 根据协议搜索 protocol="ftp" 搜索 FTP 服务
country 根据国家代码搜索 country="CN" 搜索位于中国的资产
region 根据地区（省）搜索 region="Zhejiang" 搜索位于浙江的资产
city 根据城市搜索 city="Hangzhou" 搜索位于杭州的资产
app 根据识别出的应用搜索 app="Microsoft-Exchange" 搜索 Microsoft Exchange 服务器
banner 根据服务 banner 信息搜索 banner="ssh-2.0" 搜索 SSH 服务
os 根据操作系统搜索 os=windows 搜索 Windows 系统的资产
icp 根据 ICP 备案号搜索 icp="京ICP备123456号" 搜索指定备案号的资产
jsmith 根据 JS 文件中的内容搜索 jsmith="apiKey" 搜索 JS 文件中泄露的 API 密钥
fid 根据 Fid 搜索 fid="k/aeVZYqZmCzy4+qixGnqw==" 搜索特定 Fid（文件唯一标识）的资产
逻辑运算符
运算符 说明 示例
= 等于（精确匹配，区分大小写） header="server: nginx"
== 完全等于（精确匹配，完全一致） server=="nginx/1.18.0"
!= 不等于 os!=windows
&& 与（同时满足） title="login" && body="admin"
= / ~ 正则匹配（高级功能，通常需要会员） title~".*登录.*"
= / \* 模糊匹配（包含） title*"admin"(等同于 title="admin")
搜索实例

1. 基础搜索

  • 查找特定公司资产：domain="douyin.com"
  • 查找标题中包含“登录”的页面：title="登录"
  • 查找某个 IP 段（CIDR格式）：ip="123.123.123.0/24"

2. 组合搜索（常用技巧）

  • 查找日本的 WordPress 站点：app="WordPress" && country="JP"
  • 查找中国境内开放 3389 端口的 Windows 服务器：port="3389" && country="CN" && os=windows
  • 查找某后台登录页面，但排除 404 页面：title="后台管理" && body!="404"
  • 查找 ThinkPHP 框架且存在 phpinfo 信息的站点：app="ThinkPHP" && body="phpinfo()"

3. 漏洞影响范围评估

  • 查找某特定组件或版本（例如 Apache Log4j 漏洞）：app="Apache-Log4j" && after="2021-12-01"(查找某时间后更新的资产，可能包含漏洞版本)header="x-apache-log4j"(查找特定的响应头)
  • 查找某特定漏洞的利用特征：body="weblogic" && title="500" && body="java.lang.NullPointerException"

4. 资产发现与梳理

  • 查找某公司的所有子域名：domain="xxx.com"
  • 查找所有使用了某证书的资产（常用于发现子域名或兄弟公司）：cert="xxx.com"
注意事项
  1. 转义特殊字符：如果搜索的值中包含空格、引号等特殊字符，需要用 ```` 进行转义。例如：title="\"系统登录\""
  2. 使用括号：复杂的逻辑组合建议使用括号 ()来明确优先级。例如：(title="admin" || title="登录") && country="US"
  3. is_honeypot：判断是否是蜜罐（非绝对准确，但很有参考价值）。例如，在搜索时可以考虑排除蜜罐：app="WordPress" && is_honeypot=false
  4. after/ before：按时间筛选资产，格式为 YYYY-MM-DD。例如：app="Redis" && after="2023-01-01"（查找 2023 年后更新的 Redis 服务）
  5. 统计语法：使用 fid字段和统计功能可以快速汇总资产。例如，先搜索 domain="fofa.info"，然后使用右侧的统计功能，可以快速看到这个域名下开放了哪些端口、使用了哪些服务器等。
  6. API 调用：上述所有语法都可以通过 FOFA 的 API 进行调用，实现自动化的资产监控和漏洞扫描。

OneForAll{主动}

需要3.6 及以上版本的python

配置

许多模块（如搜索引擎爬取、证书透明度查询）需要配置 API 密钥才能达到最佳效果，否则很容易被限流。

在config/setting.py下，去相应的网站申请 API Key，然后填入。

  • SSL 证书查询：需要配置 censys_api_id和 censys_api_secret。
  • 搜索引擎爬取：推荐配置 google_api_key和 google_cx_id(可自定义搜索引擎)，以及 shodan_api_key, fofa_api_email, fofa_api_key, zoomeye_api_key等。
  • DNS 解析服务器：可以配置 resolver_nameservers为更快的公共 DNS（如 114.114.114.114, 8.8.8.8）。
常用参数
参数 全称 说明 示例
--target 必选。指定要收集的目标域名，可以是单个或多个（用逗号分隔）。 --target sina.com.cn
--targets 从文件中读取目标域名列表，一行一个。 --targets ./domains.txt
--brute 启用子域名爆破。默认不开启，因为速度较慢。 --brute True
--dns --dns True 启用 DNS 解析验证子域名是否存活，并解析出 IP。默认开启，通常不需要手动设置。
--port --port True 对解析出的 IP 进行常用端口扫描。默认不开启。 --port True
--req --req True 启用 HTTP 请求，获取标题、状态码、Banner 等信息。默认开启。
--takeover --takeover True 检查子域名劫持（CNAME 记录指向已失效的云服务等）。 --takeover True
--fmt --format 指定结果输出格式，支持 txt, csv, json, yaml, html, xls, xlsx, dbf, latex。 --fmt json
--path 指定结果保存的目录，默认为 ./results/。 --path ./my_results/
--alive --alive True 在 TXT 文本格式输出中，仅输出存活的子域名。 --alive True
--help 查看完整的帮助信息。 python3 oneforall.py --help
使用示例

1. 快速收集

标准综合收集（最常用），启用所有接口查询和验证，并进行简单的 HTTP 请求获取 Web 信息。

2. 深度收集

综合收集 + 子域名爆破。注意：这会非常耗时。

3. 批量扫描

4. 深度扫描 + 端口扫描

在综合收集和爆破的基础上，同时对发现的 IP 进行端口扫描。

5. 指定输出格式

将结果输出为 JSON 和 Excel 格式，便于后续自动化处理。

生成的subdomains.txt只包含解析成功的子域名，适合直接用于后续的渗透测试
