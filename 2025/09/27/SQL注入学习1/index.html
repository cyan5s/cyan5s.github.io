updatexml是 MySQL 中的一个 XML 处理函数，它的标准语法是：

  • xml_target：要修改的 XML 文档（可以是字符串或 XML 数据）。【数字】
  • xpath_expr：XPath 表达式，用于定位要修改的节点。
  • new_value：替换后的新值。【数字】

关键点：

  • 第二个参数 xpath_expr是 concat(0x7e, (select database()))，即 ~security。

  • ~（0x7e）在 XPath 语法中是非法字符，MySQL 会抛出 XPATH 语法错误：

0x7e和 ~的关系

0x7e是 ASCII 编码中 ~（波浪号/tilde）的十六进制表示

为什么 SQL 注入要用 0x7e而不用 '~'？

  1. 避免引号被过滤：
    • 如果 SQL 语句过滤了单引号 '，直接写 '~'可能会被拦截。
    • 0x7e不需要引号，可以绕过过滤
  2. 某些数据库或 WAF（防火墙）可能对字符串 '~'做特殊处理，但 0x7e通常不会被拦截。
  3. 报错注入中，0x7e能确保生成一个非法 XPath 表达式，而 '~'可能被误解析。

为什么使用ASCII码而不直接写's':

1. SQL语法限制

if()函数的条件部分通常需要返回一个布尔值（True/False）或数值，而直接比较字符可能会导致语法错误或逻辑失效。某些数据库（如MySQL）在特定配置下，字符比较可能被隐式转换，导致结果不可靠。

2. 避免引号被转义或过滤

  • 如果目标系统对**单引号（'）**做了转义或过滤，直接写 's'会导致Payload被破坏：

3. 兼容性更强

ASCII码是通用的数值表示，适用于所有数据库（MySQL、PostgreSQL、Oracle等），而字符 's'可能因数据库的字符集或排序规则（Collation）产生歧义。

  • 某些数据库区分大小写（'s'≠ 'S'），而ASCII码可以精确控制。
  • 二进制模式下，字符比较可能不如数值比较稳定。

4. 防御规避

WAF（Web应用防火墙）通常会检测单引号包裹的字符串（如 's'），但可能忽略纯数字（如 115）。使用ASCII码能绕过部分规则：
