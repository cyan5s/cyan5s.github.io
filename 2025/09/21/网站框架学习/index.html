【1】前端基础

html&css&js

技术类型 主要功能 在代码中的表现形式
HTML 定义网页结构和内容 所有标签（如 <div>, <script>, <style>）及其属性（如 id, class, src）
CSS 定义网页元素的样式和外观 style属性内的代码 和 <style>标签内的代码
JS (JavaScript) 定义网页的行为和交互逻辑 <script>标签内的代码 或 通过 src属性引入的外部脚本

【2】PHP

基本特点

  • 服务器端执行：PHP 代码在服务器上运行，生成 HTML 后发送给浏览器，用户看不到 PHP 源码。
  • 通常运行在 Apache 或 Nginx 服务器上。支持多种数据库，常用于构建 CMS、电商网站、论坛等。
  • 开源

基础语法

  • 通常嵌入在 HTML 中，以 <?php开头，以 ?>结尾。

  • 注释：

    • 单行注释：//或 #

    • 多行注释：/* ... */

  • 变量以 $开头，无需声明类型（动态类型语言）

  • 条件判断（if-else）

  • 循环（for / while）

  • 数组（Array）

【4】简单动态网站（html&php&sql）

前端

html&php

动态网站的部分前端代码是后端php动态生成的html，由echo或print输出，为动态HTML。

内容类型 占比 特点
静态HTML 30-50% 页面框架、UI结构、固定内容
动态HTML 50-70% 数据展示、条件内容、循环结构

大多数开发者会套用模板

后端

数据库&php

浏览器请求->php接受请求->php脚本链接数据库->php脚本构造SQL查询语句->数据库执行查询->php处理结果->生成html->返回浏览器

前后端交互

流程

浏览器请求 → 服务器接收 → PHP处理 → 数据库操作 → PHP生成HTML → 服务器响应 → 浏览器渲染

主要方式

表单提交（同步）

用户登录、注册、数据提交

特点：

  • 页面会刷新
  • 数据通过HTTP body传输
  • 适合关键数据操作

AJAX请求（异步）

实时搜索、无刷新点赞、动态加载

特点：

  • 页面无刷新
  • 数据通常使用JSON格式
  • 适合频繁的小数据交互

Cookie/Session机制

用户状态保持

数据流向：

PHP: setcookie() → HTTP响应头 → 浏览器存储 → 后续请求自动携带

URL参数与重定向

分页、筛选、简单状态传递

数据格式与传输

格式 适用场景 PHP处理方式
FormData 表单提交 $_POST/$_GET
JSON API接口 json_decode(file_get_contents('php://input'))
XML 传统Web服务 SimpleXML扩展
